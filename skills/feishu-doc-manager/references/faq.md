# 常见问题解答

## 权限相关问题

### Q: 提示 "没有权限访问该文档" 怎么办？
**A:**
1. 先运行 `feishu_app_scopes()` 检查当前权限
2. 确认是否有 `docx:document` 或 `wiki:wiki` 权限
3. 对于文档，需要把机器人添加为文档协作者
4. 对于知识库，需要在知识库空间设置中添加机器人为成员
5. 如果是租户级权限，需要管理员在开放平台申请

### Q: 如何给机器人添加文档权限？
**A:**
1. 打开飞书文档
2. 点击右上角"分享"
3. 搜索机器人名称（DocAgent）
4. 添加为协作者，设置"可编辑"权限
5. 确认保存

### Q: 知识库操作失败，提示权限不足？
**A:**
1. 进入知识库空间
2. 点击空间设置
3. 选择"成员管理"
4. 添加机器人为成员
5. 设置适当的角色权限（管理员或编辑者）

## 文档操作问题

### Q: 如何从 URL 中提取 doc_token？
**A:**
```
文档 URL 格式：https://example.feishu.cn/docx/doxcnABC123XYZ
doc_token = "doxcnABC123XYZ"（/docx/ 后面的部分）
```

### Q: 创建的文档在哪里可以找到？
**A:**
1. 创建文档时可以指定 `folder_token` 放到特定文件夹
2. 如果不指定，会放到机器人的云盘根目录
3. 可以通过 `feishu_drive(action="list")` 查看文件
4. 创建成功后会返回文档链接，直接访问即可

### Q: 如何批量操作多个文档？
**A:**
1. 先使用 `feishu_drive` 列出所有需要操作的文档
2. 逐个进行操作
3. 注意飞书 API 的速率限制，建议适当延迟
4. 记录每个操作的结果，便于检查

### Q: write 操作会覆盖整个文档吗？
**A:**
是的，`write` 操作会替换文档的全部内容。如果你只是想添加内容：
- 使用 `append` 操作在文档末尾追加
- 使用 `insert` 操作在指定位置插入
- 或者先 `read` 读取内容，修改后再 `write`

## PDF 处理问题

### Q: 扫描版 PDF 能识别吗？
**A:**
- 文字版 PDF 可以直接提取文本
- 扫描版 PDF 需要 OCR 识别
- 可以使用 pytesseract、easyocr 等 OCR 工具
- 识别准确率取决于图片清晰度和字体

### Q: PDF 识别后的格式很乱怎么办？
**A:**
- 先提取纯文本，然后重新整理格式
- 利用模型能力重新组织内容结构
- 对于复杂表格，可能需要手动调整
- 可以要求用户提供源文件而不是扫描版

### Q: 英文 PDF 翻译后格式丢失？
**A:**
- 翻译时尽量保持原文结构
- 使用 Markdown 格式保留标题、列表等
- 重要文档可以生成中英对照版本
- 翻译后重新应用格式规范

## 合同模板问题

### Q: 如何管理多个合同模板？
**A:**
1. 使用多维表格（Bitable）建立模板目录
2. 每个模板保存为独立的飞书文档
3. 在 Bitable 中记录模板信息和链接
4. 按类型、行业等分类管理
5. 定期更新和维护模板版本

### Q: 生成合同时变量没有替换？
**A:**
- 检查变量占位符格式是否正确（使用 `{{变量名}}`）
- 确认所有变量都有对应的值
- 可以先列出模板中的所有变量，逐个确认
- 建议使用标准变量名，避免拼写错误

### Q: 合同模板可以加密吗？
**A:**
- 飞书文档本身支持权限控制
- 可以设置只有特定人员能访问
- 敏感合同建议单独存放，严格控制权限
- 定期检查和更新访问权限

## 多维表格问题

### Q: 如何创建一个合同模板管理表格？
**A:**
1. 使用 `feishu_bitable_create_app` 创建应用
2. 创建表格，添加以下字段：
   - 模板名称（文本）
   - 模板类型（单选）
   - 文档链接（URL）
   - 创建日期（日期）
   - 版本号（文本）
   - 状态（单选）
3. 开始使用

### Q: Bitable 中的日期格式怎么设置？
**A:**
- 日期时间字段使用毫秒级时间戳
- Python 中可以用 `int(time.time() * 1000)` 获取
- 也可以直接传入日期字符串，某些情况下会自动转换

### Q: 如何从 Bitable 中查询特定记录？
**A:**
- 先使用 `feishu_bitable_list_records` 列出所有记录
- 在内存中过滤需要的记录
- 或者建议用户在飞书中使用视图和筛选功能

## 格式问题

### Q: Markdown 转飞书文档格式不正确？
**A:**
- 飞书文档支持标准 Markdown，但某些扩展语法可能不支持
- 复杂表格建议简化或使用图片
- 数学公式可能需要单独处理
- 可以先创建测试文档验证格式

### Q: 如何统一文档格式？
**A:**
- 参考 [format-standards.md](format-standards.md) 中的规范
- 创建文档时就应用标准格式
- 现有文档可以使用格式规范重新整理
- 建立模板文档，复用格式

### Q: 图片插入后显示不正常？
**A:**
- 确保图片链接可以公开访问
- 或者先上传图片到飞书云盘
- 检查图片格式是否支持（PNG、JPG 等）
- 大图片建议压缩后再插入

## 其他问题

### Q: 操作失败了怎么办？
**A:**
1. 先查看错误信息，了解失败原因
2. 检查权限是否足够
3. 确认参数格式是否正确
4. 检查网络连接
5. 可以重试一次，或者换个方式操作
6. 如果持续失败，记录错误信息，联系技术支持

### Q: 可以撤销操作吗？
**A:**
- 飞书文档有历史版本功能，可以在网页中恢复
- 删除操作比较危险，建议先确认再执行
- 重要操作前可以先备份文档内容
- 使用 `write` 前先 `read` 保存当前内容

### Q: 如何提高工作效率？
**A:**
- 建立常用模板，减少重复工作
- 使用 Bitable 管理资源，便于查找
- 批量操作时先规划，再执行
- 记录常用操作的步骤和参数
- 定期整理和归档文档

### Q: 有什么最佳实践吗？
**A:**
1. 重要文档先备份再修改
2. 使用清晰的命名规范
3. 定期整理文件夹结构
4. 建立文档索引便于查找
5. 敏感文档严格控制权限
6. 定期检查和更新模板
